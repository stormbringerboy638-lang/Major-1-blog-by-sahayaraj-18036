Study of RIP

Routing Information Protocol: The Distance Vector Standard

What is RIP?

The Routing Information Protocol (RIP) is one of the oldest Distance Vector routing protocols. It employs the hop count as a routing metric. RIP prevents routing loops by implementing a limit on the number of hops allowed in a path from source to destination.

Visualizing node updates in a dynamic network

Key Characteristics

  • 1
    Algorithm: Uses the Bellman-Ford algorithm to calculate the best route.
  • 2
    Metric: Hop Count. The maximum number of hops allowed is 15. A hop count of 16 is considered infinite (unreachable).
  • 3
    Updates: Routing updates are broadcasted periodically every 30 seconds.

Timers in RIP

To maintain stability, RIP uses several timers:

Update Timer
30 Seconds
Frequency of router updates.
Invalid Timer
180 Seconds
Time before a route is marked invalid.

Limitations

"Count to Infinity" Problem

RIP suffers from slow convergence. When a link fails, it takes time for the network to stabilize, leading to the "Count to Infinity" problem where loops occur. Techniques like Split Horizon and Route Poisoning are used to mitigate this.
